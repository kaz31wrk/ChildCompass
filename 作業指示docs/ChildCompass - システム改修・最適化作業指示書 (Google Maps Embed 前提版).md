# ChildCompass - システム改修・最適化作業指示書 (Google Maps Embed 前提版)  
本書は、育児支援アプリ「ChildCompass」のバックエンド（Google Apps Script: GAS）およびフロントエンド（React/HTML）における課題の解決、セキュリティ向上、UX改善を行うための具体的な作業指示書です。 ※ユーザー様の要望に基づき、お出かけ地図は **Google Maps Embed (iframe)** をダイナミックに活用する設計へと最適化しています。  
## 1. 現状のシステム課題とレビュー結果  
## 1.1 バックエンド（Code.js）の課題  
1. **グローバルスコープでのアクティブスプレッドシート取得による潜在的バグ**  
    * 行頭の const SS = SpreadsheetApp.getActiveSpreadsheet(); は、GASのトリガー実行やコンテキストによっては null を返す可能性があり、動作が不安定になる原因となります。  
    * **対策**: スプレッドシートの取得を安全なゲッター関数 getSS_() にカプセル化します。  
2. **PropertiesService の頻繁な呼び出しによるクォータ（制限）消費**  
    * initSpreadsheet はリクエストのたびに実行されますが、その中で毎回 PropertiesService.getScriptProperties().getProperty を呼ぶのはGASの制限に達するリスクがあります。  
    * **対策**: APIキーの取得は必要な関数内でのみ行い、プロパティが存在しない場合の初期化ロジックを最適化します。  
3. **致命的な未定義関数：geocodeAddress_**  
    * handleAction に case 'geocodeAddress' が定義されているものの、バックエンドに該当する実体関数が存在しません。これにより住所検索時にエラーが発生します。  
    * **対策**: Maps.newGeocoder() を用いた住所から緯度経度への変換処理を実装します。  
4. **データベース（シート）書き込み時の排他制御（ロック）の欠如**  
    * 複数ユーザーの同時アクセスや、ボタンのダブルタップによる連続送信時、データの衝突や不整合が発生するリスクがあります。  
    * **対策**: LockService を導入し、データの追加・更新処理に排他ロックを適用します。  
## 1.2 フロントエンド（index.html）の課題  
1. **Google Maps Embed の「動的表示切り替え」の未活用**  
    * 従来は現在地の1箇所を表示する固定的な使い方に留まっており、お出かけ周辺検索で見つかった複数のスポットを地図に反映できていませんでした。  
    * **対策**: お出かけ施設リストから「任意の施設」をタップした際、または「GPS現在地」を取得した際に、Google Maps Embedの iframe src を動的に切り替えるリアクティブな地図コンポーネントを開発します。  
    * 検索ワードとして「施設名＋住所」をエンコードしたセキュアな無料埋め込みURL形式 maps.google.com/maps?q=...&output=embed を採用し、APIキー漏洩リスクを完全に排除します。  
2. **APIキーのセキュリティリスク**  
    * GASから getGeminiPromptAndKey などでフロントエンドに APIキー を直接返却する仕組みを維持しつつ、万が一キーが未設定（プレースホルダー状態）である場合にも、アプリが停止しない自動モックフォールバックや親切な警告 UI を導入します。  
3. **モバイル対応 UI/UX のさらなる強化**  
    * スマホ端末での操作時に、カテゴリチップや記録日時入力のタップターゲット、入力フォームが押しづらい、または崩れる問題がありました。  
    * **対策**: Tailwind CSS の構造を調整し、ビューポート幅に柔軟に適応する、よりモダンで洗練されたモバイルフレンドリー UI に刷新します。  
## 2. 各ファイルの改修・実装要件  
## 【要件 1】バックエンド（Code.js）の堅牢化  
* グローバル変数 SS を廃止し、セーフゲッター getSS_() を実装。  
* 書き込みを伴うすべてのアクション（addLog_, addGrowth_, getOrCheckUser_, addMilestone_ など）に LockService.getScriptLock() を適用し、最大10秒間の書き込み競合防止を施す。  
* geocodeAddress_(params) を新規実装し、フロントエンドからの住所検索に完全対応する。  
## 【要件 2】お出かけマップ（index.html 内 FacilityMap）の動的埋め込み移行  
* 親コンポーネントから selectedFacility (選択中の施設) または userLocation (現在地) を受け取り、それらに基づいて最適な Google Maps Embed 検索URLを算出・更新する。  
* 施設リスト内の「地図で表示 📍」ボタンをタップすると、その位置へ埋め込みマップがスムーズにフォーカス（リロード）されるようにする。  
* Leaflet.js 依存コードを綺麗に削除し、地図初期化失敗エラーや表示崩れのトラブルを完全に排除する。  
## 【要件 3】デザインシステムの統合とモバイルファースト UI/UX の向上  
* 日時選択（<input type="datetime-local">）や数値セレクトボックスがはみ出さないよう、box-sizing と w-full を徹底。  
* おむつや体調、食事ログを直感的に記録できる入力フィールドの微調整、お出かけ情報に「電話確認モーダル」を完全連動。  
## 3. 開発および検証手順（テスト仕様）  
1. **GASへのデプロイ**:  
    * Code.js と MilestonesData.js をGASエディタに貼り付け、保存します。  
    * ウェブアプリとして新しいバージョンで「デプロイ」を実施し、アクセス権限を「全員」に設定します。  
2. **フロントエンドお出かけマップの動作テスト**:  
    * 端末で index.html を開き、「お出かけ」タブを開きます。  
    * 周辺の公園や小児科が一覧表示されている中から任意の施設を選び、「地図で表示 📍」ボタンをタップします。  
    * 地図がその施設の名前と位置ピンに切り替わることを確認します。
